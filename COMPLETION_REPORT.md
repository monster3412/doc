## Итоговый отчёт: Интеграция энергии/времени в WorkshopReport

### Задача
Внедрить поддержку энергетических и временных метрик в приложение WorkshopReport, чтобы они:
1. Загружались из пользовательских файлов
2. Обрабатывались вместе с основными метриками (план/факт)
3. Отображались в UI как дополнительные KPI
4. Экспортировались во все форматы

### Выполненные работы

#### 1. Расширение функции агрегации `_enrich()` [ui.py]
**Было:** обработка 5 столбцов (workshop, date, product, plan_qty, fact_qty)
**Стало:** обработка 9-11 столбцов с сохранением energy/time данных
```python
# Новая структура строки после _enrich():
[ws, dt, pr, plan, fact, dev, pct, plan_energy, fact_energy, plan_time, fact_time]
```
- Агрегирует по (workshop, date, product) как раньше
- Теперь также суммирует энергию и время по группам

#### 2. Новые расчётные функции [calculator.py]
```python
calculate_energy_intensity_plan()    # Плановая энергоёмкость
calculate_time_intensity_plan()      # Плановая трудоёмкость
calculate_time_deviation()           # Отклонение по времени
build_efficiency_table()             # Полная таблица эффективности
```
Каждая функция:
- Проверяет наличие требуемых колонок
- Корректно обрабатывает нулевые значения (избегает division-by-zero)
- Возвращает результат или 0.0/пустую таблицу при ошибке

#### 3. Новый фрейм "Эффективность" в UI [ui.py]
**Добавлено:**
- `efficiency_frame` с 7 новыми метриками
- `efficiency_labels` словарь для динамического обновления
- Метод `_update_efficiency_summary()` для пересчёта

**Отображаемые метрики:**
- 📊 Энергоёмкость факт (кВт·ч/шт)
- 📊 Энергоёмкость план (кВт·ч/шт)
- 📊 Отклонение энергии (кВт·ч)
- ⏱️ Трудоёмкость факт (ч/шт)
- ⏱️ Трудоёмкость план (ч/шт)
- ⏱️ Отклонение времени (ч)
- 📈 Использование времени (%)

#### 4. Встроенный датасет с энергией/временем [ui.py]
**Было:** 180 строк x 5 колонок (BUILTIN)
**Стало:** 180 строк x 9 колонок
- Каждая строка теперь содержит реалистичные значения energy/time (~0.5 кВт·ч за единицу, ~0.05 ч)
- Данные согласованы: энергия примерно в 10 раз больше времени (в часах)

#### 5. Обновлённые экспорты
- **CSV**: добавлены колонки `план_энергии`, `факт_энергии`, `план_времени`, `факт_времени`
- **JSON**: добавлены поля в объект данных
- **HTML**: новая таблица сводки эффективности с интенсивностями и статусом цеха

#### 6. Интеграция с сохранением обратной совместимости
- Если файл содержит только 5 столбцов → работает как раньше (без energy/time)
- Если файл содержит энергию/время → дополнительные метрики активны
- Встроенный датасет автоматически использует energy/time

### Как использовать

#### Встроенный датасет (по умолчанию)
- При запуске приложения автоматически загружается BUILTIN с энергией/временем
- Нажать "Сформировать" → видят KPI + таблицу эффективности

#### Собственные файлы
1. Подготовить CSV/Excel с колонками: `plan_qty`, `fact_qty`, `plan_energy`, `fact_energy`, `plan_time`, `fact_time`
2. Импортировать через "Загрузить"
3. Нажать "Сформировать"
4. Энергетические метрики появятся автоматически

#### Синонимы колонок
Приложение распознаёт: план/факт, план_количество/факт_количество, plan/fact, plan_qty/fact_qty и т.д.
