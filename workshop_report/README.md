# WorkshopReport

WorkshopReport — это готовое декстопное приложение для анализа производственных планов и фактических показателей по цехам. Оно помогает быстро сравнивать план и факт, видеть отклонения, оценивать выполнение производственных задач и анализировать эффективность использования энергии и времени.

## Для чего это нужно

Приложение подходит для руководителей производства, начальников цехов, планировщиков и аналитиков, которым важно оперативно отвечать на вопросы:

- Выполнен ли план по выпуску продукции?
- Где есть отклонения по месяцам и по продуктам?
- Насколько эффективно используются энергия и рабочее время?
- Какие цеха и продукты требуют внимания?

## Что умеет продукт

### 1. Импорт и работа с данными
- Принимает данные из CSV и XLSX файлов.
- Поддерживает загрузку плановых и фактических данных.
- Работает с встроенным демонстрационным набором данных.
- Поддерживает расширенные показатели: план/факт по энергии и времени.

### 2. Аналитика по плану и факту
- Показывает план, факт, отклонение и процент выполнения.
- Предоставляет сводную картину по выбранным цехам, периодам и продуктам.
- Поддерживает фильтрацию по нескольким параметрам сразу.

### 3. KPI и сводные показатели
- Отображает основные производственные метрики в удобных карточках.
- Включает показатели эффективности:
  - энергоёмкость,
  - трудоёмкость,
  - использование времени.
- Позволяет быстро оценивать, где производственная эффективность выше или ниже ожидаемой.

### 4. Визуализация
- Отображает графики по выполнению плана и факта.
- Поддерживает различные виды диаграмм для анализа по месяцам и по продукции.
- Помогает визуально увидеть тренды и отклонения.

### 5. Экспорт результатов
- Экспортирует отчёты в CSV, JSON, HTML и Excel.
- Сохраняет как табличные данные, так и графики.
- Подходит для передачи результатов руководству, коллегам или в архив.

## Как выглядит рабочий процесс

1. Загрузите файл с планом и файл с фактом или воспользуйтесь встроенными примерами.
2. Выберите нужные цеха, период и продукт.
3. Нажмите «Сформировать».
4. Изучите сводку, таблицу и графики.
5. При необходимости экспортируйте результат в нужный формат.

## Что доступно в интерфейсе

## Обработка ошибок и ограничения

Приложение умеет сообщать пользователю о проблемах, которые могут возникнуть при работе с файлами:

- отсутствующие обязательные колонки,
- некорректный формат дат,
- неподдерживаемый тип файла,
- пустые или несоответствующие по структуре данные,
- несовпадение количества строк между загружаемыми файлами.

Такие ситуации не приводят к «тихому» сбою: пользователь получает понятные подсказки и может быстро исправить входные данные.

## Алгоритм загрузки и проверки файлов

При импорте файл проходит несколько этапов проверки:

- Путь ищется сразу в нескольких местах: по заданному пути, в папке `data`, в запасной папке `workshop_report_1/data` и в текущей рабочей директории.
- Поддерживаются форматы CSV, XLSX и JSON.
- Имена колонок нормализуются: пробелы обрезаются, регистр игнорируется, русские и латинские варианты базовых столбцов распознаются.
- Колонки `workshop`, `date` и `product` обязательны, а `plan_qty` / `fact_qty` читаются как числовые значения.
- Расширенные колонки `plan_energy`, `fact_energy`, `plan_time`, `fact_time` также считываются, если присутствуют.
- Значения чисел очищаются от лишних символов, пробелов и нечисловых знаков, запятые и точки конвертируются корректно.

Если файл содержит лишние колонки, система не ломается: данные по неизвестным полям не используются в расчётах, а обязательные метрики всё равно строятся.

## Как выбирается и сортируется отчёт

При формировании отчёта строки группируются по ключам `workshop/date/product` и агрегируются по плану и факту.

- Внутренний порядок строк определяется по дате, а затем по продукту.
- Это позволяет видеть отчёт в стабильном, хронологическом порядке.
- Фильтры по цехам, периоду и компонентам применяются до построения таблицы.

## Почему важна синхронизация план/факт

Факт и план должны сравниваться по одним и тем же ключам. Если в файле отсутствует строка для конкретного цеха/месяца/продукта, система не скрывает её:

- используется `outer join`, поэтому строки с планом без факта или с фактом без плана остаются в отчёте.
- значения по отсутствующей стороне заполняются нулями для арифметики, но пользователь видит проблему прямо в таблице.
- это предотвращает скрытые искажения, когда данные в одном файле отличаются по структуре от другого.

Если структура планового файла отличается от фактического, приложение предупреждает об этом и даёт возможность проверить данные.

## Важные алгоритмические моменты

- Сводка строится на агрегировании `plan_qty` и `fact_qty` по ключу `workshop/date/product`.
- Процент выполнения считается как `fact / plan * 100`.
- При плановом количестве 0 и фактическом 0 считается, что процент выполнения равен `0.0`, чтобы не получать ошибку деления на ноль.
- Энергоёмкость рассчитывается как `sum(fact_energy) / sum(fact_qty)`.
- Трудоёмкость рассчитывается как `sum(fact_time) / sum(fact_qty)`.
- Использование времени считается как `sum(plan_time) / sum(fact_time) * 100`.
- Если расширенные метрики не заданы, отчёт всё равно строится по основным показателям, а дополнительные графики и экспорт остаются отключены.

## Краткое резюме

WorkshopReport — это практический инструмент для ежедневного производственного контроля: от загрузки данных до готового отчёта с понятной визуализацией и возможностью экспорта. Он помогает быстрее принимать решения, видеть отклонения и фиксировать проблемные зоны в производственном процессе.
